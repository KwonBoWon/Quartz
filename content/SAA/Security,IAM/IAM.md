# IAM

## IAM 개념

* AWS리소스에 대한 접근 제어와 권한 관리
* 사용자, 그룹, 역할관리
* 정책(JSON)기반 관리(최소 권한 원칙)
* MFA 지원

## IAM 주요 권한

1. 사용자 (User)
	- AWS 리소스에 액세스 권한이 있는 개별 엔티티
	- 각 사용자는 고유한 이름으로 식별
	- 프로그래밍 방식 액세스를 위해 액세스 키 제공 가능
2. 그룹(Group)
	- 동일한 권한을 공이하는 IAM 사용자들의 모음
	- 그룹에 정책을 부여하여 동일한 권한 할당
3. 역할 (Role)
	- 특정 리소스를 액세스하는 데 필요한 권한을 임시로 부여받을 수 있는 앤티티
	- 사용자가 아닌 AWS 서비스, 애플리케이션 다른 계정등에 사용 가능(EC2, Lambda)
	- 임시 자격 증명으로 안전하게 액세스
4. IAM 정책(Policy)
	- 리소스에 대한 액세스 권한을 정의하는 JSON형식의 문서
	- 허용또는 거부를 명시

## 베스트 프랙티스

1. 최소 권한 원칙
	- 꼭 필요한 권한만 부여 
2. IAM 루트 사용자 제한 사용
	- 루트 계정 을 IAM으로 대체
3. MFA 활성화
4. 정기적인 자격 증명 감사
	- 사용하지 않는 IAM사용자, 정책, 액세스 키 삭제
5. 정책 템플릿 사용
