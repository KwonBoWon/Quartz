# EC2

## EC2 개념

* 인스턴스라고 불리는 가상화 서버 제공

## EC2 종류

|**요금 모델**|**설명**|**사용 사례**|
|---|---|---|
|온디맨드 (On-Demand)|사용한 만큼만 요금 부과, 초기 비용 없음|단기 프로젝트, 테스트 환경|
|예약 인스턴스 (Reserved)|1년 또는 3년 단위로 선결제, 최대 75% 비용 절감|EC2, 구성 변경 불가|
|스팟 인스턴스 (Spot)|미사용 용량을 저렴한 가격에 제공, 90% 저렴할 수 있음|가변적인 워크로드, 배치 작업|
|컴퓨팅/인스턴스 세이빙 플랜 (Savings Plans)| 일정 시간 사용을 약정 |Lambda, Fargate가능 구성 변경 가능|

- 온디맨드: 사용한 만큼 요금 부과, 초기 비용 없음(단기 프로젝트, 테스트 환경)
- 예약 인스턴스:  EC2특정 유형을 1년또는 3년동안 예약해서 할인(특정 인스턴스, 운영체제, 리전만 적용)
	- Convertible RI를 사용하면 인스턴스 유형은 변경 가능 (할인율이 낮아짐)
	- 예약 호스트: 물리적 서버 단독으로 사용 호스트
- 스팟 인스턴스: 남는 EC2제공 최대 90%할인(사전 예고후 종료될 수 있음)(중단돼도 괜찮은 작업)
- 세이브 플랜
	- EC2 세이브 플랜: 특정 인스턴스 유형과 리전에만 적용
	- 컴퓨팅 세이브 플랜: 모든 인스턴스 유형 및 리전에 적용(EC2, Fargate, Lambda)

## EKS

## AutoScaling

* 용량 정책에 따라 자동으로 확대/축소

### 동적 스케일링

- 지속적인 수요 변화에 따라 인스턴스 수 조정
- Target Tracking Scaling: 지정한 메트릭(CPU사용량, 트래픽)에 맞춰 목표값을 설정하고 이를 유지하게 인스턴스를 확장하거나 축소
- Step Scaling: 메트릭이 특정 임계값을 초과할 때 정핸진 단계로 인스턴스를 추가하거나 제거하는 방식
- Simple Scaling: 주어진 트리거에 따라 단일 인스턴스를 추가하거나 제거

### 예측 스케일링

- 과거 데이터를 분석하여 미래의 수요를 예측하고 미리 인스턴스를 조정
- 시간대나 일정에 따른 트래픽 패턴을 예측하여 예측된 부하에 따라 인스턴스 수를 조정

## 사례

### 네트워크 성능, 저지연 사례

- 클러스터 전략 배치 그룹 생성
	- 하드웨어 및 네트워크를 최적화
	- 여러 EC2 인스턴스가 같은 물리적 서버에 위치하여 저지연과 높은 네트워크 대역폭
	- 고속 데이터 전소, 분산 애플리케이션
