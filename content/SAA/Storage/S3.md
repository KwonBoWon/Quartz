# S3

## S3 개념

- 확장 가능, 안정적, 안전한 객체 스토리지
- IAM 정책, 버킷 정책, ACL(액세스 제어 목록)으로 정교한 권한 설정
- 버전 관리, 수명주기 관리, 객체 잠금 기능

## 스토리지 종류

- Standard
	- 자주 액세스하는 데이터
- Standard-IA
	- 백업, 재해복구, 장기 보관 및 액세스 빈도 낮은
- One Zone-IA
	- 단일 가용영역, 재히 복구 불필요
- Intelligent-Tiering
	- 자동 계층화, 비용 최적화
- Glacier Instant Retrieval
	- 저비용, 즉시 액세스 가능
- Glacier Flexible Retrieval
	- 저비용, 3~5시간 내 복구 가능
- Glacier Deep Archive
	- 최저 비용, 12~48시간 내 복구 가능
- Reduced Redundancy
	- 낮은 내구성

## 액세스 제어, 암호화

- IAM 정책
	- AWS 사용자 및 역할에 대한 권한 부여
- 버킷 정책
	- 특정 버킷에 대한 액세스 제어
- ACL
	- 객체 및 버킷에 대한 권한 설정
- 서버측 암호화
	- SSE-S3: 관리 키로 암호화
	- SSE-KMS: KMS로 암호화
	- SSE-C: 고객이 제공한 키로 암호화
- 클라이언트 측 암호화
	- 데이터를 업로드 하기 전에 암호화하여 저장

- 새로운 데이터를 암호화
	- 기본 버킷 암호화
- 기존 데이터를 암호화 
	- 복사 후 암호화
- 둘 다 필요
	- 기본 버킷 암호화를 설정한 후 기존 객체에 대한 복사 작업 수행

## 사용사례

- 백업 및 복구(Glacier, Deep Archive)
- 콘텐츠 배포, 호스팅

## 베스트 프랙티스

1. 최소 권한 원칙
	- IAM 정책과 버킷 정책을 설정
2. 수명 주기 정책 사용
	- 오래된 데이터를 저비용 스토리지로 이동
3. 암호화 활성화 
	- 민감한 데이터 SSE 또는 클라이언트 암호화
4. 버전 관리 활성화
5. 모니터링 및 로깅 활성화
	- CloudWatch, CloudTrail로 추적
