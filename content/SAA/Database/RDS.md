# RDS

## RDS 개념

- 관리형 관계형 데이터베이스 서비스(RDBMS)
- 자동 백업, 복제, 고가용성, 스케일 업/다운(수동 확장)
- 다중 AZ 인스턴스 가능(동일한 리전)

## 사용사례

- 소규모 , 중규모 워크로드
- 특정 벤더의 DBMS를 그대로 사용해야 할 때

## Proxy

1. 연결 풀링(Connection Pooling)
	- 많은 애플리케이션이 짧은 주기로 연결을 해제하면 성능 저하 및 과부하고 발생할 수 있습니다.
	- 연결 풀을 관리하여 빈번한 연결/해제 오버헤드를 줄임
2. 자동 장애 조치(Failover) 가속화
	- 자동으로 활성 인스턴스를 대체하는 기능을 가속
	- RDS 장애 조치를 수 초 내에 완료
3. 보안 향상(AWS Secrets Manager 통합)
	- Secrets Manager와 통합하여 자격증명을 자동으로 관리
	- IAM 기반 인증 지원을 통해 사용해 권한을 중앙 집중적으로 제어
4. 스케일링 및 성능 최적화
	- 대량의 연결을 처리하고 자동으로 확장할 수 있어, 애플리케이션 성능 극대화
