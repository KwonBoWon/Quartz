# VPC

## VPC 개념

* 온프레미스와 AWS 클라우드 간의 보안 연결을 설정

### VPC Gateway, 연결

VPC가 외부 네트워크 또는 다른 AWS 서비스간에 연결을 하기 위한 구성요소

1. 인터넷 게이트웨이(Internet Gateway)
	- VPC 내부 리소스를 인터넷과 연결(완전 관리형)
	- 퍼블릭 IP또는 탄력적 IP를 가진 리소스를 인터넷에 연결
	- 아웃바운드, 인바운드 트래픽 지원
	- 구성
		1. 인터넷 게이트웨이를 생성하고 VPC에 연결
		2. 라우트 테이블에서 특정 서브넷 트래픽을 인터넷 게이트웨이로 라우팅
		3. 인스턴스에 퍼블릭 IP를 할당하여 인터넷 액세스 가능

2. NAT 게이트웨이(Network Address Translation Gateway)
	- 프라이빗 서브넷에 있는 리소스를 인터넷과 연결(완전 관리형)
	- 아웃바운드 트래픽만 허용(외부에서 프라이빗 서브넷으로 액세스 불가)
	- 퍼블릭 NAT 게이트웨이는 인터넷 게이트웨이와 연결되어야 함
	- 데이터베이스 또는 애플리케이션 서버가 업데이트 혹은 외부 API연결
	- 구성
		1. NAT 게이트웨이를 퍼블릭 서브넷에 배치
		2. 프라이빗 서브넷의 라우트 테이블에서 아웃바운드 트래픽을 NAT 게이트웨이로 설정

3. 가상 프라이빗 게이트웨이(Virtual Private Gateway)
	- VPC와 온프레미스 또는 다른 클라우드를 연결하기 위한 게이트웨이
	- Direct Connect 또는 VPN과 함께 사용
	- 데이터 암호화로 안전한 연결 제공
	- 구성
		1. 가상 프라이빗 게이트웨이를 생성하고 VPC에 연결
		2. Direct Connect 또는 VPN 연결을 설정
		3. 라우트 테이블에 온프레미스 네트워크로의 경로 추가
4. 트랜짓 게이트웨이Transit Gateway
	- 여러 VPC 및 온프레미스 네트워크를 중앙에서 연결할 수 있는 게이트웨이
	- 대규모 네트쿼으 아키텍쳐에서 트래픽 라우팅 단순화
	- VPN, Direct Connect, VPC 피어링 등을 하나의 TGW에서 관리 가능
5. VPC 피어링(VPC Peering)
	- 두 개의  VPC를 직접 연결하는 네트워크 방식
	- 인터넷 게이트웨이 없이 VPC간의 통신 가능
6. 게이트웨이 로드 밸런서(Gateway Load Balancer)
	- 네트워크 트래픽을 분석하고 **어플라이언스**로 전달
	- 보안 서비스와 트래픽 검사를 위한 네트워크 로드 밸런싱 가능
7. PrivateLink
	- VPC간의 프라이빗 연결을 제공하는 서비스
	- 퍼블릭 IP없이 AWS 네트워크 내부에서 통신 가능
	- VPC Endpoint 서비스를 활용하여 특정 서비스만 PrivateLink를 통해 노출 가능
	- 사례
		- SaaS또는 특정 내부 서비스(API)를 특정 VPC에 안전하게 공유
8. VPN
	- VPN을 이용하여 다른 VPC(혹은 온프레미스)와 연결 가능
	- 인터넷을 통해 암호화된 터널을 통해 연결
9. Direct Connect
	- 온프레미스와 AWS를 전용선으로 연결
	- Transit Gateway와 연동하여 여러 VPC함께 연결 가능
	- 고속, 저지연(VPN보다 빠름)
	- 연결 수립까지 2주이상 소요

### VPC Endpoint

1. 인터페이스 엔드포인트 -> 여러 서비스 내부연결
- AWS PrivateLink 기반으로 VPC내에서 대부분의 서비스에 대한 프라이빗 네트워크 연결을 제공한다.(API GW, SSM, Lambda, SNS)
- ENI를 생성하여 VPC 서브넷 내부 IP로 접근 가능
- NAT,인터넷 게이트웨이 없이 AWS 서비스에 접근 가능
- AWS 내부망을 통해 전달됨(트래픽 비용 부과)

2. 게이트웨이 엔드포인트 -> S3, DynamoDB무료
- VPC 라우트 테이블에 엔트리를 추가하여 특정 서비스(S3, DynamoDB)와의 연결 설정
- NAT, 인터넷 게이트웨이 없이 접근 가능
- AWS 내부망을 통해 전달(트래픽 요금 없음)
